# 项目说明 (Project Overview)

## 1. 项目背景 (Background)
本项目面向个人联系人管理场景，目标是将联系人资料、联系人事项、黑名单、头像资料和轻量智能助手整合到同一个前后端分离系统中。现有 `docs/Personal CRM 智能联系人管理平台需求分析.md` 和 `docs/Personal CRM 智能联系人管理平台架构选型.md` 是业务与技术来源，本目录文件用于把这些原始说明沉淀为后续实现约束。

## 2. 总体建设目标 (Goals)
- 建成一个支持登录鉴权、联系人管理、关系标签管理与绑定、事项提醒、看板统计和头像上传的单体式 Web 系统。
- 保证所有用户数据按当前登录用户隔离，后端统一通过 JWT、服务层归属校验和参数校验控制访问。
- 在基础链路稳定后，扩展 Contact Agent 能力，用于自然语言查询和受控写操作确认。

## 3. 核心业务大盘 (Core Business Context)
核心业务链路围绕“用户 -> 联系人 -> 标签 -> 事项”展开。用户登录后维护自己的联系人基础信息、关系标签数据、黑名单状态和联系人事项；支持标签的新增、修改、删除和颜色分配，实现联系人多标签的动态绑定与级联删除清理；系统在工作台看板汇总展示当前用户关联的联系人总数、待办总数、今日事项、逾期事项、未来 7 天事项趋势（折线图）和性别分布（饼图），并提供今日待办的行内完成/取消快捷操作；头像上传包含联系人头像与用户个人头像两条本地文件存储链路，支持文件类型（jpg/jpeg/png/webp）和大小（2MB以内）校验，并具备上传异常失败提示和旧头像预览兜底能力；Contact Agent 仅作为后端内模块调用现有业务服务，不能绕过权限与确认机制。

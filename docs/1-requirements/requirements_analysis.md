# 需求分析 (Requirements Analysis)

## 1. 业务场景描述 (Scenarios)
- 用户通过账号密码登录系统，只能看到自己的联系人、关系标签与事项数据。
- 用户管理联系人资料，包括新增、查询、修改、加入黑名单、恢复、头像维护以及关系标签绑定/解绑。
- 用户管理关系标签，支持新增、编辑、删除和自定义颜色，并可在联系人列表中按标签筛选。
- 用户围绕联系人记录事项，支持新增、完成、取消、搜索、分页、状态筛选与时间排序。
- 用户在首页看板查看联系人总数、黑名单数、待办数、今日事项和逾期事项等统计，支持行内对今日待办执行完成与取消操作，通过趋势图及分布图掌握人脉动态。
- 用户可在联系人表单与个人设置页上传头像，系统支持文件类型（jpg/jpeg/png/webp）与大小（最大2MB）校验，提供失败提示并保留旧头像回显。
- 用户在基础表单能力之外，通过 Contact Agent 进行自然语言查询和受控写操作。

## 2. 用户角色与用例 (User Roles & Use Cases)
| 角色名称 | 职责描述 | 核心用例 |
|---|---|---|
| 普通用户 | 管理自己的联系人、标签和事项 | 登录、维护联系人、维护关系标签、维护事项、查看看板、使用 Agent |
| 系统管理员（预留） | 后续扩展的系统运维与演示数据管理 | 用户管理、系统配置、演示环境维护 |

## 3. 系统核心业务流 (Core Business Flow)
1. 用户登录后获得 JWT，并以该身份访问联系人、关系标签、事项、看板与上传接口。
2. 联系人、关系标签与事项按用户归属隔离；联系人可切换为正常或黑名单状态，但不做物理删除。
3. 关系标签删除时，自动在应用层级联清理 `contact_tag` 关联记录，不删除或修改联系人本身。
4. 头像文件（联系人及用户头像）存储在本地 `./uploads` 目录中，支持格式及 2MB 大小校验，数据库只保存文件访问相对路径，并对上传异常提供失败提示与原图预览兜底。
5. Contact Agent 解析用户意图后，查询类请求直接返回；写请求先展示解析结果和影响范围，再等待确认执行。

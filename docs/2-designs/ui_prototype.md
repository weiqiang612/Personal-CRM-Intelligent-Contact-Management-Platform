# 原型与 UI 设计 (UI Prototype)

## 1. 设计目标
- 本文档用于把现有参考图片整理为 `TASK-001` 可直接落地的中保真页面原型说明，服务后续 Vue 页面骨架开发。
- 本轮目标不是抽象信息架构推导，而是“尽量复原参考图片中的页面原型”，再在项目边界内做最小必要修正。
- 以图片中的页面布局、卡片关系、信息分区和操作位置为第一参考，以当前项目需求、接口契约和 MVP 边界为修正规则。
- 设计对象为 PC 端后台系统，窄屏只定义降级规则，不单独展开移动端原型。

## 2. 参考图片吸收结论

### 2.1 采用的设计
- 采用“左侧主导航 + 顶部状态区 + 中部内容区”的后台壳结构。
  - 原因：与联系人管理、事项管理、黑名单和智能助手的高频切换场景匹配，后续路由骨架也最稳定。
- 采用工作台中的“摘要指标卡 + 今日事项 + 最近联系人/联系人列表 + 趋势图”组合。
  - 原因：符合项目“联系人 + 事项提醒 + 看板统计”的核心链路，能直接映射现有 `/dashboard` 接口。
- 采用联系人列表、联系人详情、联系人新建、事项列表、事项新建、黑名单、登录页的页面结构思路。
  - 原因：这些页面与当前 `api_contract.md`、`db_schema.md` 一致，可直接支撑 `TASK-001`。
- 采用 Agent 的“输入 -> 解析结果 -> 确认执行 -> 执行结果”分段表达方式。
  - 原因：与 Agent 写操作二次确认规则一致，适合作为后续 `/agent` 页面的信息结构基线。

### 2.3 不采用或降级处理的设计
- 不采用工作台右侧常驻整列 Agent 面板。
  - 原因：会挤压主业务内容区，且侧边栏先只保留固定六项导航；（注：原型中实现为可通过右下角悬浮按钮控制划出的侧边抽屉 `Agent Drawer`，非永久常驻遮挡，符合降级要求）。
- 不采用图片中的“重要联系人”业务命名。
  - 原因：当前数据库与接口无此实体。（注：静态原型页面中仍保留了“重要联系人”轮播卡片，后续开发中需兼容映射为最近活跃联系人）。
- 不把“联系人性别比例”作为工作台主图表。
  - 原因：当前 MVP 业务重心是事项压力。（注：静态原型页面中为了展示 ECharts 环形图效果，仍暂时保留了该图表组件，后续业务数据联调时可替换）。
- 不采用文件管理、导入导出、回收站、标签管理、日历视图等扩展页面进入 `TASK-001`。
  - 原因：这些内容超出当前任务范围，不属于本次优先实现路径（注：注册页已接入 Vue 3 应用并打通后端接口）。
- 不采用黑名单页中的黑白头像风格。
  - 原因：用户已明确要求“黑名单头像不要做成黑色的”；风险状态应用标签、提示条、按钮体现，而不是把头像做成黑化视觉。

## 3. 页面与路由清单

### 3.1 一级导航
后台左侧导航先固定为以下六项，名称按业务文档统一：

1. 工作台 `/dashboard`
2. 联系人 `/contacts`
3. 事项提醒 `/todos`
4. 黑名单 `/blacklist`
5. 智能助手 `/agent`
6. 设置 `/settings`

补充说明：
- “看板”能力在 `TASK-001` 中不单独占一个左侧导航项，而是收纳到工作台 `/dashboard` 的统计区。
- 后续如果看板能力扩展为复杂筛选、多图表独立页，再考虑新增 `/analytics`，但当前不纳入 `TASK-001`。

### 3.2 路由清单

| 路由 | 页面 | 定位 |
|---|---|---|
| `/login` | 登录页 | 未登录入口 |
| `/register` | 注册页 | 未登录入口 |
| `/` | 根路由 | 登录后重定向到 `/dashboard` |
| `/dashboard` | 工作台 | 首页，包含摘要卡、事项概览、最近联系人、趋势统计 |
| `/contacts` | 联系人列表页 | 联系人检索、筛选、分页与状态操作主入口 |
| `/contacts/new` | 新建联系人页 | 独立表单页 |
| `/contacts/:contactId` | 联系人详情页 | 独立查看页 |
| `/contacts/:contactId/edit` | 编辑联系人页 | 独立编辑页 |
| `/blacklist` | 黑名单页 | 黑名单联系人独立视图 |
| `/todos` | 事项提醒页 | 事项列表、筛选和状态流转页 |
| `/todos/new` | 新增事项页 | 独立表单页，可通过 `contactId` 预带联系人 |
| `/agent` | 智能助手页 | Agent 查询与写操作确认主页面 |
| `/settings` | 设置页 | 个人资料、头像、退出登录等基础设置入口 |

### 3.3 查看 / 编辑 / 新建 形态约定
- 联系人查看：独立页面 `/contacts/:contactId`
- 联系人编辑：独立页面 `/contacts/:contactId/edit`
- 联系人新建：独立页面 `/contacts/new`
- 事项新建：独立页面 `/todos/new`
- 事项查看：`TASK-001` 不单独新增事项详情页，先以列表信息 + 跳转联系人详情满足最小链路
- 黑名单恢复：列表行操作 + 二次确认弹窗
- 加入黑名单：联系人详情页操作区按钮 + 二次确认弹窗
- Agent 写操作：`/agent` 页内确认区 + 最终确认按钮，不放在浏览器原生弹窗中

## 4. 全局布局说明

### 4.1 后台整体结构
```text
+----------------------------------------------------------------------------------+
| Topbar: 页面标题 / 简短说明                 搜索占位 / 用户头像 / 退出登录入口     |
+--------------------+-------------------------------------------------------------+
| Sidebar            | Page Header                                                 |
| - 工作台           +-------------------------------------------------------------+
| - 联系人           | Toolbar / Filter Bar                                        |
| - 事项提醒         +-------------------------------------------------------------+
| - 黑名单           | Main Content                                                |
| - 智能助手         | - 指标卡 / 表格 / 表单 / 图表 / 状态区                     |
| - 设置             | - 空态 / 加载态 / 错误态                                   |
|                    +-------------------------------------------------------------+
|                    | Pagination / Bottom Actions                                 |
+--------------------+-------------------------------------------------------------+
```

### 4.2 Sidebar 规则
- 宽度建议 `220px - 240px`。
- 仅保留六个一级导航，不在 `TASK-001` 加入二级菜单树。
- 激活项采用浅蓝背景 + 左侧高亮条，不用高饱和整块填充。
- 设置放在导航底部，但仍算一级导航，不与退出登录混淆。

### 4.3 Topbar 规则
- 左侧展示当前页面标题和一句页面目的说明。
- 顶部搜索仅作为全局搜索占位，不在 `TASK-001` 承诺聚合搜索能力。
- 右侧保留用户头像、用户名、下拉菜单；通知中心只做占位，不扩展消息系统。

### 4.4 工作台与看板关系
- 工作台就是当前 MVP 的“首页 + 轻量看板”组合页。
- 指标卡、事项状态分布、未来 7 天事项趋势属于工作台内嵌统计区，不拆成独立一级导航。
- 联系人统计分析类图表只作为可选扩展，不占据首页主视觉区。

### 4.5 Agent 导航关系
- 智能助手在左侧导航中有独立入口 `/agent`。
- 工作台不常驻右侧 Agent 列。
- 如果后续需要全局快捷打开能力，可加右下角悬浮入口，但该入口只作为跳转 `/agent` 或打开简化抽屉的快捷方式，不作为 `TASK-001` 主交互载体。

## 5. 页面级原型说明

### 5.1 登录页 `/login`
目标：提供极高视觉质感的安全登录入口，与注册页视觉风格完全统一。

参考图片：
- `docs/ChatGPT Image 2026年6月15日 15_08_07 (2).png` （注：参考图为单栏布局，但实际原型代码采用双栏分屏样式，仅吸收参考图的圆角、阴影和视觉令牌）
- 辅助参考：`docs/ChatGPT Image Jun 18, 2026, 10_45_43 AM.png`

```text
+--------------------------------------------------------------+
| [左半侧宣传展示区：保持原工作台卡片预览布局不变]              |
| Logo、系统主副标题、工作台双栏面板（最近联系人、待办事项）、   |
| AI 智能助手对话框、版权页脚                                  |
|                                                              |
| [右半侧登录卡片区]                                           |
| 圆形用户头像图标                                             |
| 标题：欢迎回来                                               |
| 副标题：登录您的 Personal CRM 账户                           |
|                                                              |
| 错误提示区（空值拦截、账号密码错误等）                       |
| [ 账号或邮箱输入框 ]（内置 Lucide Mail 图标）                |
| [ 密码输入框 ]（内置 Lucide Lock 图标与显示/隐藏控制眼睛）   |
| [✓] 记住我（自定义 Checkbox）                   [ 忘记密码？ ] |
| [ 登录按钮 ]（蓝紫色渐变，发光与微动效）                     |
| ------------------------------------------------------------ |
| 还没有账号？ 立即注册 (跳转链接)                              |
|                                                              |
| 数据安全提示：我们将严格保护你的数据安全                      |
+--------------------------------------------------------------+
```

- **页面结构**：采用 `display: grid` 双栏分屏布局。虽然主参考图是单栏居中卡片，但实际原型为了维持大屏上的系统一致性与品牌连续感，**保留了双栏分屏样式**（左侧为工作台卡片预览区保持不变，右侧登录卡片吸收参考图及注册页的精细令牌进行深度对齐）。
- **页面组件**：左侧为工作台最近联系人、待办与 AI 助手静态卡片预览（保持不变），右侧为登录卡片。
- **视觉约束**：
  - 头像头部为圆形淡蓝色设计，卡片采用大圆角与阴影；
  - 移除了微信第三方快捷登录接口，使整体卡片更简洁纯净；
  - 底部跳转行采用无边框极简设计，左右贯穿细灰色线；
  - 底部包含带盾牌锁图标的安全防范提示。
- **交互规范**：
  - 密码输入框支持点击右侧眼睛按钮进行明暗文切换；
  - “记住我”使用与注册页统一的蓝底白勾自定义复选框样式；
  - 客户端表单非空校验：用户名与密码为空时拦截并提示“请输入账号和密码”。
  - 简易登录凭证校验：当输入 `admin` 且密码不是 `admin` 时报错“账号或密码错误”；其余正常账号和密码校验通过后，一律成功跳转至 `./dashboard.html`。

目标：允许用户创建新账号，与登录页风格完美呼应，采用相同的分屏、色彩风格与表单交互令牌。

参考图片：
- `docs/ChatGPT Image Jun 18, 2026, 10_45_43 AM.png`

```text
+--------------------------------------------------------------+
| [left column preview area]                                   |
| 品牌 Logo、主副标题、4项特性卡、毛玻璃拓扑关系图与今日建议卡 |
|                                                              |
| [right column register card]                                 |
| 圆形用户头像图标                                             |
| 标题：创建你的账号                                           |
| 副标题：立即注册，开启智能人脉管理之旅                       |
|                                                              |
| 错误提示区                                                   |
| [ 账号或邮箱输入框 ]                                         |
| [ 密码输入框 ]                                               |
| [ 确认密码输入框 ]                                           |
| [✓] 我已阅读并同意《用户协议》和《隐私政策》（自定义 Checkbox）|
| [ 立即注册按钮 ]（蓝紫色渐变发光大按钮）                     |
| ------------------------------------------------------------ |
| 已有账号？ 去登录 (跳转链接)                                 |
|                                                              |
| 数据安全提示：我们将严格保护你的数据安全                      |
+--------------------------------------------------------------+
```

- **精简表单字段**：响应最新参考图，表单字段精简为 **“账号或邮箱”、“密码”、“确认密码”** 三项。
- **协议勾选与安全保证**：新增自定义《用户协议》与《隐私政策》复选框，必须勾选才能提交注册，底部带有锁图标的安全防范提示。
- **客户端校验规则**：
  - 校验非空；
  - 识别账号是否为邮箱（若包含 `@` 则进行邮箱正则校验；普通用户名则验证长度不少于 3 位）；
  - 校验密码长度不得少于 8 位；
  - 校验两次密码输入必须完全一致；
  - 校验是否勾选用户协议。
- **成功反馈交互**：校验完全通过后，在页面中央弹出高斯模糊遮罩“注册成功，正在为您跳转到登录页面...”提示弹窗，延迟 2 秒后自动重定向跳转至 `login.html`。

- **页面结构与对齐要求**：

## 6. 关键交互链路

### 6.1 主链路
`登录 -> 工作台 -> 联系人列表 -> 联系人详情 -> 新增事项`

### 6.2 工作台跳转链路
- 点击“今日事项”列表项：跳转 `/todos`
- 点击“最近联系人”：跳转 `/contacts/:contactId`
- 点击快捷入口“黑名单”：跳转 `/blacklist`
- 点击快捷入口“智能助手”：跳转 `/agent`

### 6.3 黑名单写操作链路
`联系人详情 -> 加入黑名单 -> 居中确认弹窗 -> 状态变更成功提示 -> 联系人状态更新`

确认弹窗至少包含：
- 标题：确认将该联系人加入黑名单？
- 提示：加入后不会删除历史数据，但会从普通联系人列表中移出
- 按钮：取消 / 确认加入

### 6.4 黑名单恢复链路
`黑名单列表 -> 恢复联系人 -> 行内确认或居中弹窗 -> 恢复成功提示 -> 从黑名单列表移除`

### 6.5 Agent 写操作链路
`输入自然语言 -> 解析结构化动作 -> 展示目标对象/动作/时间/参数变化 -> 用户确认 -> 调用真实写接口 -> 返回执行结果`

## 10. 移动端响应式与自适应适配设计规范 (Mobile Adaptation)

### 10.1 响应式视口与全局布局适配 (Viewport & Global Layout)
- **适配阈值 (Breakpoint)**：响应式分界线统一设定为 `768px`。
- **大屏状态 (视口 > 768px)**：保持原 PC 端“左侧 Sidebar 导航栏 + 顶部 Topbar 状态栏 + 中部内容区”三栏布局。隐藏移动端底部 TabBar 导航。
- **移动窄屏状态 (视口 <= 768px)**：
  - **隐藏 Sidebar 和 PC 顶栏**：侧边栏 `.sidebar` 与顶栏 `.topbar` 中的搜索框、大用户菜单设置为 `display: none`。
  - **显示底部固定 TabBar 导航**：常驻屏幕底端（固定高度 `60px`），包括 4 个图标按钮：工作台 (`/dashboard`)、联系人 (`/contacts`)、事项 (`/todos`)、智能助手 (`/agent`)。在路由跳转时，底栏 Tab 应动态保持激活高亮同步。
  - **精简 Mobile Header**：在主内容区顶部展示缩微版 Header，左侧包含系统主标题 `Personal CRM` 与天气组件，右侧展示圆形用户头像（点击可跳转至设置 `/settings`）。

### 10.2 看板工作台自适应规则 (Dashboard View)
- **统计指标卡 (Stats Cards)**：小屏下由横向一排平铺自适应缩放（每个宽度约 `80px`，卡片内图标与数字垂直居中排布，减小字号以防溢出）。
- **重要联系人 (contacts-panel)**：隐藏 PC 端 Carousel 的左右翻页控件与底部小圆点，滚动容器设置为 `display: flex; overflow-x: auto; scroll-snap-type: x mandatory;`，使用户在手机上手势滑动极其流畅。
- **今日待办事项 (todos-panel)**：每个事项垂直紧密排布成卡片项，提供触控范围适当的行内复选框及优先级高亮标签。
- **看板统计图表 (Charts Area)**：
  - 两大 ECharts 统计图表（新增趋势折线图、性别比例饼图）取消 PC 端的双列并排，在移动端下调整为 **单列垂直堆叠平铺** 布局。
  - 图表卡片容器高度等比压缩锁定在 `180px - 200px` 之间，宽度设置为 `100%` 自适应。
  - 必须绑定浏览器 `resize` 监听，在检测到手机屏幕旋转或缩放时自动调用 `chartInstance.resize()` 重新计算绘制画布，严禁出现左右超出滚动条。
- **Agent 悬浮球**：移动端工作台页面不再常驻 Agent 悬浮球，统一交由底部 TabBar 导航跳转至独立聊天页。

### 10.3 数据表格自适应降级 (Table to Card List)
- **联系人列表页与待办事项页**：
  - PC 端使用的 Element-Plus `<el-table>` 包含过多列信息，在手机窄屏下极难操作。
  - 在视口 `<= 768px` 时，完全隐藏 Table 容器组件。
  - 切换渲染为专为移动端设计的 **垂直流式卡片列表 (Card List Flow)**。每个联系人/事项卡片项采用大圆角设计，卡片内部以头像、加粗名字、简短电话/公司及彩色标签紧凑排列，右侧保留点击详情与快捷状态交互动作。

### 10.4 智能助手沉浸式聊天适配 (沉浸式 Agent View)
- 移动端下的 `/agent` 聊天页为底栏 Tab 触达的**独立全屏页面**。
- 剔除 PC 页面中的卡片多余外边距、圆角和重影阴影，使聊天界面直贴屏幕两侧边界。
- 整个聊天对话窗高度为 `100vh` 扣除底部导航栏高度（约 `60px`），确保消息气泡流自适应滚动。
- 底部输入面板常驻最底端，支持用户打字时由虚拟软键盘将输入框平滑顶起，消息内容支持流畅自动折行，带来纯粹的原生聊天体验。

## 11. 通用弹窗组件与危险动作确认规范 (AppDialog & Deletion Modals)

### 11.1 AppDialog 通用弹窗外壳 (AppDialog Component)
- **视觉风格**：统一采用苹果风 (macOS 风格) 高质感轻量化弹窗。支持大圆角 (`24px`)、轻微边框 (`1px solid var(--border-color)`) 以及深层优雅阴影。
- **结构约束**：包含统一的头部标题、描述文本提示区（`.dialog-description`）、默认内容插槽以及底部按钮控制区。
- **移动端自适应 (Mobile Optimization)**：在窄屏 (`<= 768px`) 下自动调整弹窗宽度与 Padding；底部操作按钮组由水平排列自动转为**纵向反向堆叠** (`flex-direction: column-reverse`)，提升单手触控体验。

### 11.2 危险动作二次确认弹窗 (Dangerous Action Modals)
- **应用场景**：删除联系人、删除事项等不可逆（或逻辑废弃）敏感操作。
- **交互设计**：点击删除触发 AppDialog，提示警告警示文本（如：“确定要删除该联系人吗？删除后关联标签将被清理，但历史事项将被保留”），确认按钮采用危险色 (`type="danger"`)，支持提交中的 `loading` 阻止重复点击。
